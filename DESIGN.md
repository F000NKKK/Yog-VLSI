# Yog VLSI — Design Document

> **Very Large Scale Integration** — проектирование, fabrication и деплой редстоун-микросхем с Rust-ускоренной симуляцией.

## 1. Обзор

Мод добавляет три ключевых блока: **VLSI Workbench** (верстак), **Microchip** (микросхема, 6 тиров), **ALU** (арифметико-логическое устройство, тоже 6 тиров), а также **аналоговые и цифровые кабели** для соединения ALU с внешним миром. Игрок проектирует редстоун-схему в виртуальном мире, «печатает» её на микросхему, вставляет микросхему в ALU — и получает сверхбыструю симуляцию своей логики в реальном мире.

Вся симуляция редстоуна выполняется нативным Rust VM — это даёт скорость до 40 тиков/с (2× vanilla) на Netherite-чипе.

## 2. Microchip (микросхема)

### 2.1 Тиры

| Тир | Материал | Тиков/с | Размер мира | Кратно vanilla |
|-----|----------|---------|-------------|----------------|
| Wood | Дерево | 5 | 16×16×16 | 0.25× |
| Stone | Камень | 10 | 32×32×32 | 0.5× |
| Gold | Золото | 20 | 64×64×64 | 1× |
| Iron | Железо | 25 | 64×64×64 | 1.25× |
| Diamond | Алмаз | 30 | 128×128×128 | 1.5× |
| Netherite | Незерит | 40 | 256×256×256 | 2× |

- Чем выше тир — тем больше виртуальный мир и быстрее симуляция.
- Каждый чип имеет уникальный UUID.
- Чип хранит метаданные в NBT (ChipMeta): ID, тир, имя, список портов.
- Схема (CircuitData) хранится на сервере через Storage API (отдельно от NBT, т.к. для чипа 256×256 это десятки тысяч блоков).

### 2.2 Порты

Порты — это ноды чипа, через которые он взаимодействует с внешним миром. В отличие от физических блоков на границе мира, порты **не привязаны к конкретным координатам** — они настраиваются в UI и существуют как логические точки подключения.

В виртуальном мире (редакторе) игрок размещает блоки `yog-vlsi:port` с NBT `{mode: "input"|"output"|"bidirectional"}` в любом месте сетки. При сохранении схемы эти блоки регистрируются как порты чипа.

```
Порт {
    label: "A0" | "CLK" | "Q" | ...
    dir:   Input | Output | Bidirectional
    pos:   (x, y, z)  — позиция внутри виртуального мира
}
```

Внутри VM каждый порт — это `Cell` с `BlockType::Port(mode)`, находящийся в конкретной ячейке грида. Сигнал на input-порту задаётся извне (ALU), сигнал с output-порта читается наружу.

Количество портов не ограничено — ограничена только сетка чипа.

## 3. VLSI Workbench (верстак)

### 3.1 Крафт

```
I S I
S C S
I R I

I = iron_ingot
S = smooth_stone
C = crafting_table
R = redstone_block
```

### 3.2 Ресурсная система (Resource Ammo)

Верстак хранит ресурсы по принципу «краски в MFU»:
- Нет лимита на хранение.
- Игрок пополняет ресурсы, кидая предметы в верстак.
- Ресурсы трекаются как `HashMap<item_id, quantity>`.
- Персистентность: Storage API, ключ = позиция блока.

### 3.3 Fabrication (печать чипа)

Стоимость печати = **25% от vanilla-рецепта** каждого блока в схеме (размеры маленькие).

Пример:
- Redstone wire: 1 redstone dust → 0.25 (округляется вверх)
- Repeater: 2 torches + 1 dust + 3 stone → 0.5 dust + 0.5 stick + 0.75 stone за блок
- И т.д.

Ресурсы списываются из хранилища верстака, чип программируется (NBT + CircuitData на сервер).

### 3.4 Virtual World Editor (проектирование)

По нажатию кнопки «Design» в GUI верстака игрок телепортируется в creative-мир размером с тир чипа:
- Плоский мир из bedrock-пола.
- Игрок размещает **аналоговые блоки портов** (`yog-vlsi:port`) в любых клетках сетки. Каждый порт настраивается: Input/Output/Bidirectional и **подпись** (label, отображается floating text над блоком).
- Игрок строит редстоун-схему из любых доступных блоков, подключая редстоун к пинам портов.
- Все энтити (pearls, item frames, etc.) запрещены и чистятся при входе/выходе.
- Кнопка «Save» → CircuitData сохраняется на сервер (с координатами и подписями всех портов), игрок возвращается.

## 4. ALU (арифметико-логическое устройство)

### 4.1 Тиры ALU

ALU тоже имеет 6 тиров, как и чипы. Тир ALU определяет:

| Тир | Материал | Макс. чипов внутри | Цифровых каналов / сторону |
|-----|----------|--------------------|----------------------------|
| Wood | Дерево | 2 | 8 |
| Stone | Камень | 3 | 16 |
| Gold | Золото | 4 | 24 |
| Iron | Железо | 5 | 32 |
| Diamond | Алмаз | 6 | 48 |
| Netherite | Незерит | 8 | 64 |

- Аналоговых портов всегда 6 (по одному на сторону), независимо от тира.
- Цифровые каналы = `цифровых_каналов × 4 цвета`. Netherite: 64×4 = 256 бит на сторону.

### 4.2 Крафты

#### ALU (базовый)

```
G C G
C R C
G D G

G = gold_ingot
C = copper_ingot
R = repeater
D = diamond
```

Крафт для более высоких тиров — замена центрального/угловых материалов на материал тира.

#### Analog Cable (аналоговый кабель)

```
R R R

R = redstone_dust
```

3 redstone dust → 3 analog cable. Прокладывается как редстоун-пыль.

#### Digital Cable (цифровой кабель)

```
G R G
R I R
G R G

G = gold_nugget
R = redstone_dust
I = iron_ingot
```

4 кабеля за крафт. Прокладывается как редстоун-пыль, может идти рядом с аналоговым без пересечений.

#### Redstone Port (блок адаптера редстоун ↔ ALU)

```
S S S
S R S
S S S

S = smooth_stone
R = redstone_dust
```

Ставится на любую сторону ALU. С одной стороны подключается аналоговый кабель, с другой — обычный редстоун.

### 4.3 Внешние порты ALU

ALU имеет **ровно 6 внешних портов** — по одному на каждой стороне блока (North, South, East, West, Up, Down). Эти порты — единственный интерфейс между ALU и внешним миром.

Внутренние порты чипов **не выходят наружу напрямую**. Игрок в UI связывает порты чипов с внешними портами ALU (или между собой — см. 4.5).

Редстоун **не взаимодействует с ALU напрямую**. Для подключения редстоуна нужен **Redstone Port** на стороне ALU и **Analog Cable**.

#### Типы внешних портов ALU

**Аналоговый (Redstone Port):**
- Один порт = один аналоговый сигнал 0–15.
- Подключается: `Редстоун → Redstone Port (блок) → Analog Cable → ALU`.
- Конфигурируется как Input или Output.

**Цифровой (Digital Port):**
- Размещается на одной стороне ALU.
- До `N` каналов (зависит от тира ALU), каждый разделён на 4 цвета (Red, Green, Blue, Yellow).
- Канал = пара `(номер, цвет)`, 1 бит каждый (ON/OFF).
- Канал `(N, цвет)` на ALU A соединяется с каналом `(N, тот же цвет)` на ALU B через **Digital Cable**.
- Netherite ALU: 64 канала × 4 цвета = 256 бит на сторону.

### 4.4 Кабели

| Кабель | Передаёт | Пропускная способность | Соединяет |
|--------|----------|------------------------|-----------|
| **Analog Cable** | Аналоговый (0–15) | 1 сигнал | Redstone Port ALU ↔ внешний редстоун |
| **Digital Cable** | Цифровой (битовый) | До 256 бит | Digital Port ALU A ↔ Digital Port ALU B |

Оба кабеля — отдельные блоки. Цифровой и аналоговый кабель могут идти по одним и тем же координатам не пересекаясь (разные типы проводов).

### 4.5 Порты в виртуальном мире (чипа)

**Все порты внутри виртуального мира — аналоговые.** В редакторе игрок ставит блок `yog-vlsi:port` в любой клетке сетки, настраивает режим (Input/Output/Bidirectional) и задаёт **подпись** (label) — например, «CLK», «A0», «Q».

Подпись отображается:
- На самом блоке порта в виртуальном мире (floating text).
- В UI ALU как имя узла.

При подключении редстоуна к порту в виртуальном мире сигнал читается VM как значение 0–15. При сохранении чипа все порты с их подписями и координатами попадают в CircuitData.

### 4.6 UI: нодовый редактор ALU

GUI ALU открывается по ПКМ и представляет собой **flow-граф** в стиле ComfyUI / n8n / Blender Nodes:

```
┌─────────────────────────────────────────────────────┐
│  ALU (Netherite)  │  Чипов: 2/8  │  Каналов: 64    │
├──────────┬────────────────────────────┬─────────────┤
│          │                            │             │
│  Входы   │     [Чип "Adder"]          │  Выходы     │
│  ALU     │     ┌─A0──┐  ┌─SUM        │  ALU        │
│          │     │     │  │             │             │
│  ◉ N     │  IN─┤     ├──┤OUT       ──│── ● N       │
│          │     │     │  │             │             │
│  ◉ S     │     └─B0──┘  └─CARRY      │  ◉ S        │
│          │                            │             │
│  ◉ E     │     [Чип "Decoder"]        │  ◉ E        │
│          │     ┌─IN──┐  ┌─Q0         │             │
│  ◉ W     │  IN─┤     ├──┤OUT       ──│── ● W       │
│          │     └─────┘  └─Q1         │             │
│  ◉ U     │                            │  ◉ U        │
│          │                            │             │
│  ◉ D     │                            │  ◉ D        │
│          │                            │             │
├──────────┴────────────────────────────┴─────────────┤
│  [+ Добавить чип]  [Авто-линковка]  [Сохранить]     │
└─────────────────────────────────────────────────────┘
```

**Элементы UI:**
- **Слева:** 6 внешних портов ALU (N/S/E/W/U/D) — перетаскиваются как узлы-источники/приёмники.
- **Центр:** установленные чипы. Каждый чип показан как прямоугольник с пинами (портами) сверху и снизу. Пины подписаны (label из виртуального мира).
- **Справа:** 6 внешних выходных портов ALU.
- **Рёбра:** направленные линии от output-пина к input-пину. Создаются перетаскиванием от пина к пину.
- **Цвета пинов:** Input = синий, Output = красный, Bidirectional = фиолетовый.
- **Слоты чипов:** внизу панель с доступными чипами из инвентаря. Перетаскивание в центр добавляет чип.

**Операции:**
- Перетащить чип из панели в центр → установить.
- Перетащить от output-пина к input-пину → создать линк.
- ПКМ по линку → удалить.
- ПКМ по пину внешнего порта → переключить Input/Output/Bidirectional.
- Кнопка «Авто-линковка» → автоматически соединить все порты с одинаковыми именами (A0→A0, CLK→CLK).

### 4.7 Tick-обработчик

Каждый серверный тик (20/с):
- Для каждого чипа в ALU — шаг VM.
- Частота шага = `tick_rate` чипа. Netherite (40) → 2 шага VM за 1 серверный тик. Wood (5) → 1 шаг VM за 4 серверных тика.
- После шага: чтение output-портов чипов → передача сигналов по внутренним линкам → обновление внешних портов ALU.
- Внешние аналоговые порты → обновление redstone-сигнала на стороне блока.
- Внешние цифровые порты → передача битов по цифровому кабелю.

### 4.8 Установка чипов

Через GUI (нодовый редактор), не через ПКМ. Игрок открывает ALU, видит панель доступных чипов в инвентаре, перетаскивает их в рабочую область.

## 5. Rust Redstone VM

### 5.1 Архитектура

```
RedstoneVM {
    tier: Tier,
    width, height: u32,
    grid: Vec<Cell>,          // width × width × height
    updates: VecDeque<Pos>,   // очередь обновлений
    tick: u64,
}
```

Каждая ячейка:
```
Cell {
    block: BlockType,
    power: 0..15,
    strongly_powered: bool,
    weakly_powered: bool,
    repeater_timer: u8,
    torch_burnout: u8,
}
```

### 5.2 Алгоритм тика

1. **Reset** — сброс power у всех проводов, сброс weakly_powered.
2. **Process updates** — обработка очереди: факелы (инверсия), повторители (задержка), observer (2-тиковый импульс).
3. **Propagation (BFS)** — от всех источников питания (факелы, повторители, рычаги, кнопки, redstone_block, target, pressure plates, trapped_chest, detector_rail, I/O-порты) → BFS по redstone_wire с затуханием -1 за блок.
4. **Solid powering** — strong power от редстоун-компонентов в solid-блоки; weak power через соседние solid-блоки.
5. **Timers** — decrement repeater_timer, torch_burnout.

### 5.3 Поддерживаемые блоки

**Редстоун:** wire, torch, wall_torch, repeater, comparator, lever, stone_button, wood_button, stone_pressure_plate, wood_pressure_plate, light_weighted_pressure_plate, heavy_weighted_pressure_plate, observer, note_block, target, lamp, redstone_block.

**Solid/проводящие:** Solid (все обычные блоки), Glass (непроводящий), piston, sticky_piston.

**Контейнеры:** chest, trapped_chest, ender_chest, shulker_box, barrel, hopper, dropper, dispenser, furnace, blast_furnace, smoker, brewing_stand.
- Инвентари не симулируются, но блоки корректно проводят/блокируют редстоун.
- Trapped chest выдаёт power = количеству «зрителей» (упрощённо).

**Механика:** slime_block, honey_block, tnt, iron_door, wood_door, iron_trapdoor, wood_trapdoor, fence_gate, rail, powered_rail, detector_rail, activator_rail.

### 5.4 План развития VM

- **Comparator logic** — compare/subtract режимы, чтение контейнеров.
- **Observer detection** — детект block updates в VM.
- **Piston push/pull** — физика движения блоков.
- **Hopper/Dropper item transfer** — симуляция предметов.
- **Noteblock** — частота от питания.

## 6. Команды (debug/тестирование)

| Команда | Описание |
|---------|----------|
| `/vlsi` | Справка |
| `/vlsi chip <tier>` | Выдать чистый чип |
| `/vlsi info` | Показать NBT чипа в руке |
| `/vlsi test <tier>` | Создать тестовый чип (redstone_block → wire → lamp + порты) |
| `/vlsi vm step` | Шаг симуляции на чипе в руке |
